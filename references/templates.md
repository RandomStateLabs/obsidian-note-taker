# Obsidian Note Templates

Comprehensive templates for different note types. Choose based on content purpose.

## Technical Documentation

### Architecture Document
```markdown
---
date: YYYY-MM-DD
status: capture
type: architecture
tags:
  - architecture
  - system-design
  - technical
---

[High-level description of the system and its purpose]

[How this system fits within the broader ecosystem]

- **Interfaces**: [APIs/connections]

### Component 2
[Repeat structure]

[Describe how data moves through the system]

- **Decision 1**: [Rationale]
- **Decision 2**: [Rationale]

[Authentication, authorization, data protection]
```

## Choosing the Right Template

| Content Type | Use Case | Template |
|-------------|----------|----------|
| System design | Documenting software architecture | Architecture Document |
| Instructions | Step-by-step guides | How-To Guide |
| Business concept | New business ideas | Business Idea |
| Project planning | Initial project thoughts | Project Brainstorm |
| Team discussions | Meeting records | Meeting Notes |
| Daily planning | Personal organization | Daily Note |
| Source material | Academic/research notes | Literature Note |
| Skill development | Learning documentation | Learning Note |

## Customization Tips

1. **Add fields as needed**: Templates are starting points
2. **Remove unnecessary sections**: Keep only what's relevant
3. **Maintain consistency**: Use similar structures for similar content
4. **Evolve templates**: Update based on what works
5. **Create variants**: Make specialized versions for recurring needs
